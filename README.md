# ZCode Account Manager

GUI-менеджер аккаунтов для [ZCode](https://z.ai) (провайдер `start-plan`, OAuth Z.ai).

Позволяет хранить несколько аккаунтов и переключаться между ними в один клик,
а также вести косметические заметки по каждому аккаунту.

> ⚠️ **Неофициальный инструмент.** Не связан с ZCode/Z.ai. Используйте на свой риск.

## Возможности

- 🟢 **Текущий аккаунт** — определяется автоматически по `user_id` из JWT в `config.json`.
- ➕ **Добавление** — сохраняет текущий залогиненный аккаунт как профиль.
- ⇄ **Переключение в один клик** — закрывает ZCode, подменяет файлы аккаунта, запускает заново.
- 🔄 **Автообновление** — каждые 5 секунд перечитывает конфиг, сам видит смену аккаунта (без перезапуска менеджера).
- 🚦 **Лимит на сегодня** — флаг 🟢 Есть / 🔴 Исчерпан. Вы помечаете вручную; в 19:00 (обновление лимитов) автоматически сбрасывается в 🟢.
- 📅 **Статус подписки** — подсветка: истекла / истекает сегодня / скоро (с учётом сброса в 19:00).
- 💾 **Бэкапы** — перед каждым переключением делается резервная копия в `data/backups/`.

## Требования

- **Windows** (использует `taskkill`, пути `%USERPROFILE%\.zcode\` и `%APPDATA%\ZCode\`).
- **Python 3.8+** с Tkinter (входит в стандартную установку Python для Windows).
- Установленный **ZCode** (путь к `ZCode.exe` определяется автоматически; если не найден — есть fallback).

## Установка

```bash
git clone https://github.com/<ваш-пользователь>/zcode-account-manager.git
cd zcode-account-manager
```

Никаких зависимостей ставить не нужно — только стандартная библиотека Python.

## Запуск

Двойной клик по `run.cmd`, либо:

```bash
python account_manager.py
```

## Как это работает

«Аккаунт» в ZCode (для `start-plan`) — это два файла:

| Файл | Содержимое |
|---|---|
| `~/.zcode/v2/credentials.json` | OAuth-токены (access_token, JWT, user_info) |
| `~/.zcode/v2/config.json` | Здесь в `provider["builtin:zai-start-plan"].options.apiKey` лежит JWT, в payload которого есть `user_id` — стабильный идентификатор аккаунта |

Менеджер:
1. Копирует эти два файла в папку профиля (`data/profiles/<id>/`).
2. При переключении — делает бэкап живых файлов в `data/backups/<timestamp>/`, затем копирует файлы выбранного профиля поверх живых.
3. Закрывает ZCode (`taskkill /F /IM ZCode.exe /T`) и запускает заново.

## Структура

```
zcode-account-manager/
├── account_manager.py     # приложение (Tkinter, без внешних зависимостей)
├── run.cmd                # запуск двойным кликом
├── README.md
├── LICENSE
├── .gitignore
└── data/                  # создаётся при первом запуске
    ├── profiles.json      # список профилей + косметические заметки
    ├── profiles/<id>/     # credentials.json + config.json каждого аккаунта
    └── backups/<ts>/      # резервные копии перед каждым переключением
```

## Как добавить аккаунты

1. Войдите в ZCode под первым аккаунтом (OAuth Z.ai, start-plan).
2. Откройте менеджер → **«+ Добавить текущий аккаунт»**.
3. Выйдите из аккаунта в ZCode, войдите под вторым → добавьте.
4. Повторите для остальных.

Менеджер различает аккаунты по `user_id`, дубли не пройдут.

## Безопасность

- ⚠️ Файлы в `data/profiles/` содержат **токены доступа** — это чувствительные данные.
- Эти папки добавлены в `.gitignore` и **не попадут в репозиторий**.
- Не коммитьте и не публикуйте свою папку `data/`.
- Бэкапы перед переключением — в `data/backups/`, можно откатиться вручную.
