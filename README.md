# ZCode Account Manager

GUI-менеджер аккаунтов для [ZCode](https://z.ai) (провайдер `start-plan`, OAuth Z.ai).

Позволяет хранить несколько аккаунтов и переключаться между ними в один клик,
автоматически регистрировать новые аккаунты Z.AI и вести косметические заметки.

> ⚠️ **Неофициальный инструмент.** Не связан с ZCode/Z.ai. Используйте на свой риск.

## Возможности

### Менеджер аккаунтов
- 🟢 **Текущий аккаунт** — определяется автоматически по `user_id` из JWT в `config.json`.
- ➕ **Добавление** — сохраняет текущий залогиненный аккаунт как профиль.
- ⇄ **Переключение в один клик** — закрывает ZCode, подменяет файлы аккаунта, запускает заново.
- 🔄 **Автообновление** — каждые 5 секунд перечитывает конфиг, сам видит смену аккаунта (без перезапуска менеджера).
- 🚦 **Лимит на сегодня** — реальные остатки по моделям через официальный billing API zcode.z.ai (без капчи).
- 📅 **Статус подписки** — подсветка: истекла / истекает сегодня / скоро (с учётом сброса в 19:00).
- 💾 **Бэкапы** — перед каждым переключением делается резервная копия в `data/backups/`.

### 🤖 Авторегистрация аккаунтов
- **Один аккаунт или пакет** — список `email:password` по строкам, обработка очередью.
- **Полный цикл автоматически**: капча (Playwright) → регистрация → подтверждение по почте (IMAP) → создание API key → OAuth ZCode → JWT.
- **Без прокси** — по умолчанию идёт напрямую с вашего IP (`direct://`). При желании можно указать residential-прокси.
- **Живой прогресс** — окно с логом каждого шага в реальном времени + галочка подробного лога.
- **Авто-имена** — новые аккаунты получают имена `autoreg 1, autoreg 2, ...` с подпиской на 4 дня.
- Каждый успешно созданный аккаунт автоматически добавляется в менеджер.

## Требования

### Для менеджера аккаунтов
- **Windows** (использует `taskkill`, пути `%USERPROFILE%\.zcode\` и `%APPDATA%\ZCode\`).
- **Python 3.8+** с Tkinter (входит в стандартную установку Python для Windows).
- Дополнительно: `pip install requests cryptography` (для billing-лимитов и шифрования `enc:v1:`).
- Установленный **ZCode** (путь к `ZCode.exe` определяется автоматически).

### Для авторегистрации (дополнительно)
- **Node.js 18+** и npm.
- **Playwright + Chromium**:
  ```bash
  npm install
  npx playwright install chromium
  ```
- **Почтовый аккаунт с IMAP** (например, Firstmail — `imap.firstmail.ltd:993`). Логин Z.AI = логин IMAP.

Двойной клик по `run.cmd`, либо:

```bash

```

(порт 3847) — отдельный запуск не требуется.

## Как это работает

### Менеджер аккаунтов

«Аккаунт» в ZCode (для `start-plan`) — это два файла:

| Файл | Содержимое |
|---|---|
| `~/.zcode/v2/credentials.json` | OAuth-токены, зашифрованные форматом `enc:v1:` (AES-256-GCM) |
| `~/.zcode/v2/config.json` | Здесь в `provider["builtin:zai-start-plan"].options.apiKey` лежит JWT, в payload которого есть `user_id` — стабильный идентификатор аккаунта |

Менеджер:
1. Копирует эти два файла в папку профиля (`data/profiles/<id>/`).
2. При переключении — делает бэкап живых файлов в `data/backups/<timestamp>/`, затем копирует файлы выбранного профиля поверх живых.
3. Закрывает ZCode (`taskkill /F /IM ZCode.exe /T`) и запускает заново.

### Авторегистрация

TS-движок в `src/` (Express-сервер) прогоняет полный цикл:
1. **Капча** — Playwright открывает Chromium, решает слайдер Aliyun.
2. **Регистрация** — POST на `chat.z.ai/api/v1/auths/signup`.
3. **Почта** — IMAP-опрос ждёт письмо подтверждения.
4. **Подтверждение** — finish_signup через браузер.
5. **Платформа** — логин на `api.z.ai`, создание API key.
6. **OAuth ZCode** — Playwright проходит логин + consent, получает JWT.

Готовый JWT + токены через мост шифрования (`zcode_encrypt.py`, формат `enc:v1:`)
собираются в файлы профиля и добавляются в менеджер.

## Структура

```
zcode-account-manager/
├── account_manager.py     # GUI-приложение (Tkinter)
├── billing.py             # опрос лимитов через billing API zcode.z.ai
├── zcode_encrypt.py       # мост: JWT → файлы ZCode (enc:v1: шифрование)
├── autoreg_client.py      # управление TS-сервером + HTTP-клиент API
├── run.cmd                # запуск двойным кликом
├── package.json           # Node-зависимости (авторег)
├── tsconfig.json
├── src/                   # TS-движок авторегистрации (Express + Playwright)
│   ├── index.ts           # HTTP API сервер
│   ├── autoreg/           # pipeline авторегистрации
│   └── zcode/             # работа с zcode.z.ai
├── README.md
├── LICENSE
├── .gitignore
└── data/                  # создаётся при первом запуске
    ├── profiles.json      # список профилей + косметические заметки
    ├── store.json         # данные авторега (аккаунты, jobs, прокси)
    ├── profiles/<id>/     # credentials.json + config.json каждого аккаунта
    └── backups/<ts>/      # резервные копии перед каждым переключением
```

## Как добавить аккаунты

### Вариант 1: вручную (как раньше)
1. Войдите в ZCode под первым аккаунтом (OAuth Z.ai, start-plan).
2. Откройте менеджер → **«+ Добавить текущий аккаунт»**.
3. Выйдите из аккаунта в ZCode, войдите под вторым → добавьте.
4. Повторите для остальных.

### Вариант 2: автоматически (авторегистрация)
1. Подготовьте почтовые аккаунты с IMAP (например, Firstmail).
2. В менеджере нажмите **«🤖 Авторегистрация»** → вкладка **«📦 Пакет»**.
3. Вставьте список `email:password` (по одной паре на строку).
4. Нажмите **«🤖 Запустить очередь»** — аккаунты создадутся и добавятся автоматически.

Менеджер различает аккаунты по `user_id`, дубли не пройдут.

## Переменные окружения (опционально)

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `CAPTCHA_HEADLESS` | `0` = видимый браузер при автореге (для отладки) | `1` (скрытый) |
| `FIRSTMAIL_IMAP_HOST` | IMAP-хост почтового провайдера | `imap.firstmail.ltd` |
| `FIRSTMAIL_IMAP_PORT` | IMAP-порт | `993` |
| `PORT` | Порт TS-сервера авторега | `3847` |

## Безопасность

- ⚠️ Файлы в `data/profiles/` и `data/store.json` содержат **токены доступа** — это чувствительные данные.
- Эти папки/файлы добавлены в `.gitignore` и **не попадут в репозиторий**.
- Не коммитьте и не публикуйте свою папку `data/`.
- Бэкапы перед переключением — в `data/backups/`, можно откатиться вручную.
